## Overview

BuddyPie uses Convex for backend functions. Functions are organized by domain in the `convex/` directory.

## Sandbox Functions

### Queries

#### sandboxes.list

List the current user's sandboxes.

```ts
import { api } from 'convex/_generated/api'

const sandboxes = await convex.query(api.sandboxes.list, {})
```

**Returns:** Array of sandbox records, most recent first, limited to 25.

#### sandboxes.get

Get a specific sandbox by ID.

```ts
const sandbox = await convex.query(api.sandboxes.get, {
  sandboxId: 'sandboxes_abc123'
})
```

**Arguments:**
| Name | Type | Required | Description |
|------|------|----------|-------------|
| `sandboxId` | `Id<sandboxes>` | Yes | Sandbox ID |

**Returns:** Sandbox record or null if not found/not owned.

### Mutations

#### sandboxes.createPending

Create a sandbox in 'creating' status.

```ts
const sandbox = await convex.mutation(api.sandboxes.createPending, {
  repoUrl: 'https://github.com/user/repo',
  repoName: 'repo',
  repoProvider: 'github',
  agentPresetId: 'general-engineer',
  agentLabel: 'General Engineer',
  agentProvider: 'openrouter',
  agentModel: 'minimax/minimax-m2.7',
  paymentMethod: 'credits',
})
```

**Arguments:**
| Name | Type | Required | Description |
|------|------|----------|-------------|
| `repoUrl` | string | Yes | Repository URL |
| `repoName` | string | Yes | Repository name |
| `repoBranch` | string | No | Branch name |
| `repoProvider` | 'github' \| 'git' | Yes | Provider type |
| `agentPresetId` | string | Yes | Preset ID |
| `agentLabel` | string | Yes | Preset label |
| `agentProvider` | string | Yes | AI provider |
| `agentModel` | string | Yes | AI model |
| `initialPrompt` | string | No | Kickoff prompt |
| `paymentMethod` | 'credits' \| 'x402' \| 'delegated_budget' | Yes | Payment method |

**Returns:** Created sandbox record.

#### sandboxes.markReady

Mark a sandbox as ready after launch completes.

```ts
const sandbox = await convex.mutation(api.sandboxes.markReady, {
  sandboxId: 'sandboxes_abc123',
  daytonaSandboxId: 'daytona-xyz789',
  previewUrl: 'https://3000-xyz.daytonaproxy01.net',
  workspacePath: '/home/daytona/repo',
  opencodeSessionId: 'session-123',
})
```

**Arguments:**
| Name | Type | Required | Description |
|------|------|----------|-------------|
| `sandboxId` | `Id&lt;sandboxes&gt;` | Yes | Sandbox ID |
| `daytonaSandboxId` | string | Yes | Daytona sandbox ID |
| `previewUrl` | string | Yes | Preview URL |
| `previewUrlPattern` | string | No | URL pattern with `{PORT}` |
| `workspacePath` | string | Yes | Workspace path |
| `opencodeSessionId` | string | No | OpenCode session ID |

**Returns:** Updated sandbox record.

#### sandboxes.markFailed

Mark a sandbox as failed.

```ts
const sandbox = await convex.mutation(api.sandboxes.markFailed, {
  sandboxId: 'sandboxes_abc123',
  errorMessage: 'Repository clone failed',
})
```

**Arguments:**
| Name | Type | Required | Description |
|------|------|----------|-------------|
| `sandboxId` | `Id&lt;sandboxes&gt;` | Yes | Sandbox ID |
| `errorMessage` | string | Yes | Error description |

**Returns:** Updated sandbox record.

#### sandboxes.remove

Delete a sandbox record.

```ts
await convex.mutation(api.sandboxes.remove, {
  sandboxId: 'sandboxes_abc123',
})
```

**Arguments:**
| Name | Type | Required | Description |
|------|------|----------|-------------|
| `sandboxId` | `Id&lt;sandboxes&gt;` | Yes | Sandbox ID |

**Returns:** null

## Billing Functions

### Queries

#### billing.dashboardSummary

Get the current user's billing dashboard snapshot.

```ts
const summary = await convex.query(api.billing.dashboardSummary, {})
```

**Returns:** Wallet snapshot, recent charges, recent ledger entries, current plan snapshot, and the current delegated-budget summary.

#### billing.sandboxUsage

Get billing usage details for a specific sandbox.

```ts
const usage = await convex.query(api.billing.sandboxUsage, {
  sandboxId: 'sandboxes_abc123',
})
```

**Arguments:**
| Name | Type | Required | Description |
|------|------|----------|-------------|
| `sandboxId` | `Id&lt;sandboxes&gt;` | Yes | Sandbox ID |

**Returns:** Billed total, wallet snapshot when available, sandbox charges, and delegated-budget summary.

#### billing.currentDelegatedBudget

Get the current active delegated budget for the user.

```ts
const budget = await convex.query(api.billing.currentDelegatedBudget, {})
```

**Returns:** Active delegated-budget record or `null`.

#### billing.listDelegatedBudgetSettlements

List recent delegated-budget settlements for the current user.

```ts
const settlements = await convex.query(
  api.billing.listDelegatedBudgetSettlements,
  {},
)
```

**Returns:** Array of recent delegated-budget settlement records.

#### billing.pricingCatalog

Get the active price catalog and billing environment metadata.

```ts
const pricing = await convex.query(api.billing.pricingCatalog, {})
```

**Returns:** Price version, launch prices by preset, runtime prices, and environment configuration including delegated-budget settings.

### Mutations

#### billing.createDelegatedBudget

Create a delegated budget record after the user completes the MetaMask setup flow.

```ts
const budget = await convex.mutation(api.billing.createDelegatedBudget, {
  contractBudgetId: 'budget-123',
  budgetType: 'periodic',
  interval: 'month',
  configuredAmountUsdCents: 10000,
  remainingAmountUsdCents: 10000,
  ownerAddress: '0x...',
  delegatorSmartAccount: '0x...',
  delegateAddress: '0x...',
  treasuryAddress: '0x...',
  delegationJson: '{"signed":"payload"}',
  delegationHash: '0x...',
  approvalMode: 'standing',
})
```

**Arguments:**
| Name | Type | Required | Description |
|------|------|----------|-------------|
| `contractBudgetId` | string | Yes | Onchain budget identifier |
| `budgetType` | 'fixed' \| 'periodic' | Yes | Budget type |
| `interval` | 'day' \| 'week' \| 'month' | No | Reset interval for periodic budgets |
| `configuredAmountUsdCents` | number | Yes | Budget limit |
| `remainingAmountUsdCents` | number | Yes | Initial remaining amount; must start at the full configured amount |
| `ownerAddress` | string | Yes | Budget owner address |
| `delegatorSmartAccount` | string | Yes | Smart-account address |
| `delegateAddress` | string | Yes | Backend delegate address |
| `treasuryAddress` | string | Yes | BuddyPie treasury address |
| `delegationJson` | string | Yes | Serialized delegation payload |
| `delegationHash` | string | Yes | Hash of the delegation payload |
| `delegationExpiresAt` | number | No | Delegation expiry timestamp |
| `approvalMode` | 'exact' \| 'standing' | Yes | Approval mode used during setup |
| `approvalTxHash` | string | No | Approval transaction hash |
| `createTxHash` | string | No | Budget creation transaction hash |

**Returns:** Stored delegated-budget record.

#### billing.applySubscriptionCreditGrant

Apply a Clerk subscription snapshot and grant credits when the subscription period is eligible.

```ts
const result = await convex.mutation(api.billing.applySubscriptionCreditGrant, {
  clerkSubscriptionId: 'sub_123',
  clerkSubscriptionItemId: 'si_123',
  status: 'active',
  planSlug: 'pro',
  planPeriod: 'month',
  periodStart: 1710000000000,
})
```

**Returns:** The updated account, the stored subscription snapshot, whether a new grant was applied, and the resulting grant record when one exists.

#### billing.syncClerkSubscriptionByClerkUserId

Sync a Clerk subscription snapshot for the current user by Clerk user ID.

This is primarily an internal billing-sync function used by the verified Clerk webhook handler.

#### billing.holdCredits / billing.captureCreditHold / billing.releaseCreditHold

These wallet mutations implement the internal credit hold lifecycle.

- `holdCredits` reserves wallet balance
- `captureCreditHold` converts a hold into a finalized charge
- `releaseCreditHold` returns a hold to available balance

They are part of the shared-wallet billing flow, not the x402 direct-payment flow.

#### billing.recordX402DirectCharge

This mutation records a finalized x402 charge after settlement on an authenticated `/api/x402/...` route.

## x402 HTTP Routes

x402 is implemented as authenticated server routes, not as a `billing.topUpWithX402` Convex mutation.

Current route surface:

- `POST /api/x402/sandboxes/create`
- `POST /api/x402/sandboxes/$sandboxId/restart`
- `POST /api/x402/sandboxes/$sandboxId/preview`
- `POST /api/x402/sandboxes/$sandboxId/ssh`
- `POST /api/x402/sandboxes/$sandboxId/web-terminal`

These routes:

- return `402 Payment Required` when the payment header is missing or invalid
- verify and settle x402 payments against the configured facilitator
- record finalized charges through `api.billing.recordX402DirectCharge`

## Server Functions

These functions run on the server and are not directly callable from the client.

### Daytona Operations

Located in `src/lib/server/daytona.ts`:

#### createOpenCodeSandbox

Create a complete OpenCode sandbox.

```ts
import { createOpenCodeSandbox } from '~/lib/server/daytona'

const result = await createOpenCodeSandbox({
  repoUrl: 'https://github.com/user/repo',
  branch: 'main',
  agentPresetId: 'general-engineer',
  agentProvider: 'openrouter',
  agentModel: 'minimax/minimax-m2.7',
  initialPrompt: 'Implement the feature',
  githubToken: 'gho_xxx',
})
```

**Arguments:**
| Name | Type | Required | Description |
|------|------|----------|-------------|
| `repoUrl` | string | Yes | Repository URL |
| `branch` | string | No | Branch name |
| `agentPresetId` | string | Yes | Preset ID |
| `agentProvider` | string | No | Override provider |
| `agentModel` | string | No | Override model |
| `initialPrompt` | string | No | Kickoff prompt |
| `githubToken` | string | No | GitHub OAuth token |

**Returns:**
```ts
{
  repoName: string
  repoProvider: 'github' | 'git'
  branch?: string
  workspacePath: string
  previewUrl: string
  previewUrlPattern: string
  daytonaSandboxId: string
  opencodeSessionId: string
}
```

#### deleteOpenCodeSandbox

Delete a Daytona sandbox.

```ts
import { deleteOpenCodeSandbox } from '~/lib/server/daytona'

await deleteOpenCodeSandbox('daytona-sandbox-id')
```

#### ensureSandboxAppPreviewServer

Start an app preview server.

```ts
import { ensureSandboxAppPreviewServer } from '~/lib/server/daytona'

const result = await ensureSandboxAppPreviewServer({
  daytonaSandboxId: 'daytona-sandbox-id',
  workspacePath: '/home/daytona/repo',
  port: 3000,
})
```

**Returns:**
```ts
{
  status: 'already-running' | 'started'
  port: number
  previewUrl: string
}
```

#### getSandboxAppPreviewLogTail

Get preview server logs.

```ts
import { getSandboxAppPreviewLogTail } from '~/lib/server/daytona'

const result = await getSandboxAppPreviewLogTail({
  daytonaSandboxId: 'daytona-sandbox-id',
  workspacePath: '/home/daytona/repo',
  port: 3000,
  lines: 50,
})
```

**Returns:**
```ts
{
  port: number
  output: string
  logPath: string
}
```

#### createSandboxSshAccessCommand

Generate SSH access command.

```ts
import { createSandboxSshAccessCommand } from '~/lib/server/daytona'

const result = await createSandboxSshAccessCommand({
  daytonaSandboxId: 'daytona-sandbox-id',
  expiresInMinutes: 60,
})
```

**Returns:**
```ts
{
  sshCommand: string
  expiresAt: number
}
```

## Preset Utilities

Located in `src/lib/opencode/presets.ts`:

### getOpenCodeAgentPreset

Get preset by ID with validation.

```ts
import { getOpenCodeAgentPreset } from '~/lib/opencode/presets'

const preset = getOpenCodeAgentPreset('general-engineer')
```

**Throws:** Error if preset ID is invalid.

### resolveOpenCodeModelOption

Resolve model option from provider/model inputs.

```ts
import { resolveOpenCodeModelOption } from '~/lib/opencode/presets'

const option = resolveOpenCodeModelOption({
  provider: 'openrouter',
  model: 'minimax/minimax-m2.7',
  fallbackProvider: 'venice',
  fallbackModel: 'minimax-m27',
})
```

### getOpenCodeModelOptionByProviderAndModel

Find model option by provider and model.

```ts
import { getOpenCodeModelOptionByProviderAndModel } from '~/lib/opencode/presets'

const option = getOpenCodeModelOptionByProviderAndModel(
  'openrouter',
  'minimax/minimax-m2.7'
)
```

## Sandbox Utilities

Located in `src/lib/sandboxes.ts`:

### normalizeSandboxInput

Validate and normalize sandbox creation input.

```ts
import { normalizeSandboxInput } from '~/lib/sandboxes'

const normalized = normalizeSandboxInput({
  repoUrl: 'https://github.com/user/repo',
  branch: 'main',
  agentPresetId: 'general-engineer',
  agentProvider: 'openrouter',
  agentModel: 'minimax/minimax-m2.7',
  initialPrompt: 'Build the feature',
  paymentMethod: 'credits',
})
```

### getWorkspacePath

Get the workspace path for a repository.

```ts
import { getWorkspacePath } from '~/lib/sandboxes'

const path = getWorkspacePath('my-repo')
// '/home/daytona/my-repo'
```

### isGitHubRepo

Check if a URL is a GitHub repository.

```ts
import { isGitHubRepo } from '~/lib/sandboxes'

isGitHubRepo('https://github.com/user/repo')  // true
isGitHubRepo('https://gitlab.com/user/repo')  // false
```

## Error Handling

### ConvexError

Backend errors use ConvexError:

```ts
import { ConvexError } from 'convex/values'

throw new ConvexError('Sandbox not found.')
```

### Client Handling

```ts
try {
  await convex.mutation(api.sandboxes.createPending, args)
} catch (error) {
  if (error instanceof ConvexError) {
    console.error(error.data)  // Error message
  }
}
```
