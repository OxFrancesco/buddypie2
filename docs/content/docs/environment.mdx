## Environment Variables

BuddyPie requires several environment variables for authentication, database, and AI provider integration.

### Required Variables

| Variable | Location | Description |
|----------|----------|-------------|
| `CLERK_PUBLISHABLE_KEY` | `.env.local` | Frontend Clerk public key |
| `CLERK_SECRET_KEY` | `.env.local` | Backend Clerk secret |
| `CLERK_JWT_ISSUER_DOMAIN` | `.env.local` | Clerk JWT issuer URL |
| `VITE_CONVEX_URL` | `.env.local` | Convex deployment URL |
| `DAYTONA_API_KEY` | `.env.local` | Daytona API key |

### Optional Variables

| Variable | Location | Default | Description |
|----------|----------|---------|-------------|
| `DAYTONA_API_URL` | `.env.local` | `https://app.daytona.io/api` | Daytona API endpoint |
| `CONVEX_SITE_URL` | `.env.local` | Auto-detected | Convex HTTP actions URL |
| `OPENROUTER_API_KEY` | `.env.local` | None | OpenRouter API key |
| `VENICE_API_KEY` | `.env.local` | None | Venice AI API key |

### Convex Environment Variables

Set these in the Convex dashboard under Settings > Environment Variables:

| Variable | Description |
|----------|-------------|
| `X402_PAY_TO_ADDRESS` | Wallet address receiving x402 payments |
| `X402_EIP712_TOKEN_NAME` | USDC token name (default: `USDC`) |
| `X402_EIP712_TOKEN_VERSION` | USDC token version (default: `2`) |

## Clerk Setup

### Application Configuration

1. Create a Clerk application at [clerk.com](https://clerk.com)
2. Enable authentication methods (Email, Google, GitHub, etc.)
3. Configure redirect URLs for your environment

### JWT Template for Convex

Create a JWT template named `convex`:

```json
{
  "audience": ["convex"],
  "issuer": "https://your-app.clerk.accounts.dev",
  "subject": "{{user.id}}",
  "claims": {
    "email": "{{user.primary_email_address}}",
    "name": "{{user.first_name}} {{user.last_name}}"
  }
}
```

### OAuth Scopes (GitHub)

For private repository access:

1. Enable GitHub OAuth in Clerk
2. Request `repo` scope for full repository access
3. Users must grant access during sign-in

## Daytona Configuration

### API Key

Generate an API key from the Daytona dashboard:

1. Sign in to [daytona.io](https://daytona.io)
2. Navigate to API Keys
3. Create a new key with appropriate scopes

### Sandbox Settings

Sandbox creation parameters:

```ts
{
  public: true,              // Public preview URLs
  autoStopInterval: 30,      // Auto-stop after 30 minutes idle
}
```

### Preview URL Pattern

```
https://{PORT}-{sandbox-id}.daytonaproxy01.net
```

## Model Provider Setup

### OpenRouter

1. Sign up at [openrouter.ai](https://openrouter.ai)
2. Generate an API key
3. Add to `.env.local` as `OPENROUTER_API_KEY`

Supported models:

| Model ID | Description |
|----------|-------------|
| `minimax/minimax-m2.7` | Default for general and frontend presets |

### Venice AI

1. Sign up at [venice.ai](https://venice.ai)
2. Generate an API key
3. Add to `.env.local` as `VENICE_API_KEY`

Supported models:

| Model ID | Description |
|----------|-------------|
| `openai-gpt-53-codex` | Coding-focused model |
| `claude-sonnet-4-6` | Large context model |
| `minimax-m27` | Default for docs preset |

### Environment Fallbacks

The system supports key aliases for flexibility:

```ts
VENICE_API_KEY -> VENICE_INFERENCE_KEY (fallback)
ZAI_API_KEY -> ZHIPU_API_KEY (fallback)
```

## Deployment Configuration

### Production Environment

For production deployment:

1. Set `NODE_ENV=production`
2. Use production Clerk keys (pk_live_*)
3. Use production Convex deployment
4. Configure production Daytona workspace

### Convex Deployment

```bash
# Deploy to production
bunx convex deploy

# Set production variables
bunx convex env set X402_PAY_TO_ADDRESS 0x...
```

### Build Configuration

```bash
# Build the application
bun run build

# Output in .output/
# - .output/server/index.mjs - Server entry
# - .output/public/ - Static assets
```

## Security Considerations

### Secrets Management

- Never commit `.env.local` to version control
- Use environment-specific secrets in production
- Rotate API keys periodically

### Key Scopes

| Key | Required Scope |
|-----|----------------|
| `CLERK_SECRET_KEY` | Full Clerk API access |
| `DAYTONA_API_KEY` | Sandbox management |
| `OPENROUTER_API_KEY` | Model inference |
| `VENICE_API_KEY` | Model inference |

### User Data

- User PII stored in Convex
- GitHub tokens stored via Clerk OAuth
- Wallet addresses stored for billing

<Callout type="warn">
The `CLERK_SECRET_KEY` has full access to your Clerk application. Protect it carefully.
</Callout>
