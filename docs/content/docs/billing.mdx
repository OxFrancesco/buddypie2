## Overview

BuddyPie currently supports three user-facing payment methods for agent actions:

- `credits` - spend from the shared BuddyPie wallet
- `x402` - pay per action with direct USDC settlement over HTTP 402
- `delegated_budget` - spend from a MetaMask smart-account budget that you preapprove

For the full implementation walkthrough, read [Agent Payments Deep Dive](/docs/agent-payments).

## Payment Methods And Internal Rails

The UI exposes payment methods. The ledger stores payment rails.

| Layer | Values |
|-------|--------|
| User-facing payment methods | `credits`, `x402`, `delegated_budget` |
| Ledger/payment rails | `clerk_credit`, `x402_direct`, `metamask_delegated`, `migration`, `manual_test` |

The important distinction is that `credits` is a wallet experience, not a single funding source. A wallet balance can contain subscription grants, migrated balances, and test credits, while `x402` and delegated budgets settle directly per action.

## Wallet Overview

Each user has a shared USD-denominated credit wallet with:

| Balance Type | Description |
|--------------|-------------|
| `availableUsdCents` | Spendable balance |
| `heldUsdCents` | Credits reserved for in-flight actions |
| `lifetimeCreditedUsdCents` | Total credits ever added |
| `lifetimeSpentUsdCents` | Total credits ever consumed |

The fastest way to inspect the current billing state is:

```ts
const summary = await convex.query(api.billing.dashboardSummary, {})
```

This returns the wallet snapshot, recent charges, recent ledger activity, the current Clerk plan snapshot, and the current delegated-budget record when one exists.

## How Charges Are Priced

BuddyPie prices billing events in whole USD cents and versions those prices with `unitPriceVersion`.

Current event categories:

| Event | Current behavior |
|-------|------------------|
| `sandbox_launch` | One-time launch charge; price depends on the selected preset |
| `preview_boot` | Fixed runtime charge when BuddyPie starts a preview server |
| `ssh_access` | Fixed runtime charge when BuddyPie provisions SSH access |
| `web_terminal` | Fixed runtime charge when BuddyPie provisions web terminal access |

The pricing catalog is exposed via:

```ts
const pricing = await convex.query(api.billing.pricingCatalog, {})
```

## Credits

`credits` uses holds and captures:

1. BuddyPie places a hold before the action starts.
2. If the action succeeds, BuddyPie captures the hold into a charge.
3. If the action fails or should not bill, BuddyPie releases the hold.

This flow applies to the shared wallet path only. It does not apply to `x402` or `delegated_budget`.

## x402

`x402` is implemented as direct per-request payment on authenticated server routes under `/api/x402/...`.

- BuddyPie returns `402 Payment Required` when payment headers are missing or invalid.
- The client pays with USDC on the configured Base network.
- BuddyPie settles and records the charge after the protected action succeeds.

This is a pay-per-action path. It does not top up the shared wallet.

## Delegated Budgets

`delegated_budget` uses MetaMask smart-account delegation:

1. The user creates a budget and signs delegation data.
2. BuddyPie validates the delegate, treasury, expiry, and smart-account state.
3. BuddyPie settles individual agent charges onchain against the remaining budget.
4. Settlement records are written alongside normal billing charges.

The active delegated-budget record can be read with:

```ts
const budget = await convex.query(api.billing.currentDelegatedBudget, {})
```

## Environment Model

| Environment | Network | Typical use |
|-------------|---------|-------------|
| `staging` | Base Sepolia | Testing and local validation |
| `production` | Base Mainnet | Live billing |

The pricing catalog also exposes the active chain configuration, x402 network identifier, and delegated-budget environment details.

## Auditability

BuddyPie keeps an audit trail through:

- `billingCharges` for finalized charges
- `creditLedgerEntries` for balance movements
- `subscriptionCreditGrants` for Clerk-funded credits
- `delegatedBudgetSettlements` for onchain delegated-budget settlements

If you need the full implementation-level audit and current caveats, see [Agent Payments Deep Dive](/docs/agent-payments).
