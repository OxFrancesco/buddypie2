## Overview

BuddyPie manages OpenCode presets to configure AI behavior for different work types. Each preset defines:

- Agent instructions and prompt
- Available skills
- Model provider and model
- Workspace bootstrap behavior

## Available Presets

| Preset ID | Label | Default Model | Purpose |
|-----------|-------|---------------|---------|
| `general-engineer` | General Engineer | OpenRouter / MiniMax M2.7 | Full-stack development |
| `frontend-builder` | Frontend Builder | OpenRouter / MiniMax M2.7 | UI and styling work |
| `docs-writer` | Docs Writer | Venice / MiniMax M2.7 | Documentation |

### General Engineer

Balanced repo analysis and implementation for full-stack product work.

**Configuration:**
- Provider: `openrouter`
- Model: `minimax/minimax-m2.7`
- Required env: `OPENROUTER_API_KEY`

**Skills:**
| Skill ID | Purpose |
|----------|---------|
| `buddypie-general-architecture` | Map architecture before cross-cutting changes |
| `buddypie-general-release-check` | Verification and handoff notes |

**Starter Prompt:**
> Inspect this repository, identify the files and systems relevant to the requested task, and start with the smallest high-confidence plan before editing. Then implement the change, verify the behavior you touched, and summarize the result with any remaining risks or follow-up work.

### Frontend Builder

UI-focused preset for React, styling, accessibility, and polish work.

**Configuration:**
- Provider: `openrouter`
- Model: `minimax/minimax-m2.7`
- Required env: `OPENROUTER_API_KEY`

**Skills:**
| Skill ID | Purpose |
|----------|---------|
| `buddypie-frontend-component-craft` | Component architecture and state |
| `buddypie-frontend-ui-polish` | Responsive behavior and accessibility |

**Starter Prompt:**
> Audit the relevant UI surfaces in this repository, identify the files that control the experience, and propose the smallest high-confidence frontend plan before editing.

### Docs Writer

Documentation-focused preset for guides, READMEs, and onboarding content.

**Configuration:**
- Provider: `venice`
- Model: `minimax-m27`
- Required env: `VENICE_API_KEY`

**Skills:**
| Skill ID | Purpose |
|----------|---------|
| `buddypie-docs-structure` | Plan documentation structure |
| `buddypie-docs-qa` | Verify docs against code |

**Workspace Bootstrap:**
- In BuddyPie-prepared docs sandboxes, clones Fumadocs reference to `sources/fumadocs`
- Scaffolds Fumadocs app in `docs/` or `docs-site/`
- Uses `tanstack-start` template with Bun for the prepared docs app
- Falls back to installed Fumadocs packages and the official Fumadocs docs when `sources/fumadocs` is not present
- Verifies docs changes with `bun run types:check`, `bun run build`, and a local docs server before handoff

**Starter Prompt:**
> Review the current repository, identify the documentation gaps that matter most, and build a complete Fumadocs docs pass for this project.

## Model Providers

### OpenRouter

OpenRouter provides access to multiple AI models through a unified API.

**Configuration:**
- Provider ID: `openrouter`
- Required env: `OPENROUTER_API_KEY`

**Available Models:**

| Model ID | Label | Description |
|----------|-------|-------------|
| `minimax/minimax-m2.7` | MiniMax M2.7 | Balanced general work |

**Model ID Format:**
OpenRouter models use the format `{provider}/{model}` in the OpenCode config.

### Venice AI

Venice provides uncensored AI models with privacy focus.

**Configuration:**
- Provider ID: `venice`
- Required env: `VENICE_API_KEY`

**Available Models:**

| Model ID | Label | Description |
|----------|-------|-------------|
| `openai-gpt-53-codex` | GPT-5.3 Codex | Coding and tool use |
| `claude-sonnet-4-6` | Claude Sonnet 4.6 | Large context for analysis |
| `minimax-m27` | MiniMax M2.7 | Docs workflow default |

**Model ID Format:**
Venice models use the format `venice/{model}` in the OpenCode config.

## Configuration Reference

### Preset Definition

```ts
type OpenCodeAgentPresetDefinition = {
  id: string                    // Unique identifier
  label: string                 // Display name
  description: string           // Short description
  defaultModelOptionId: string  // Default model option
  provider: string              // AI provider
  model: string                 // Model identifier
  requiredEnv: string[]         // Required environment variables
  agentPrompt: string           // Base agent prompt
  instructionsMd: string        // Instructions file content
  starterPrompt: string         // Default kickoff prompt
  starterPromptPlaceholder: string  // Input placeholder text
  skills: OpenCodeManagedSkill[]    // Available skills
  mcp: Record<string, OpenCodeManagedMcp>  // MCP servers
  workspaceBootstrap?: OpenCodeWorkspaceBootstrap  // Bootstrap config
}
```

### Model Option Definition

```ts
type OpenCodeModelOptionDefinition = {
  id: string              // Unique identifier
  label: string           // Display name
  description: string     // Short description
  provider: string        // Provider ID
  providerLabel: string   // Provider display name
  model: string           // Model identifier
  modelLabel: string      // Model display name
  requiredEnv: string[]   // Required environment variables
}
```

### Skill Definition

```ts
type OpenCodeManagedSkill = {
  id: string              // Skill identifier
  name: string            // Skill name
  description: string     // When to use
  content: string         // Skill markdown content
  permission?: 'allow' | 'ask' | 'deny'  // Execution permission
}
```

### Workspace Bootstrap

```ts
type OpenCodeDocsWorkspaceBootstrap = {
  kind: 'fumadocs-docs-app'
  sourceRepoUrl: string      // Reference repository URL
  sourceRepoBranch: string   // Git branch
  sourceRepoPath: string     // Target path in workspace
  docsTemplate: 'tanstack-start'  // Fumadocs template
  preferredDocsPath: string  // Preferred docs directory
  fallbackDocsPath: string   // Fallback docs directory
  packageManager: 'bun' | 'npm'  // Package manager
}
```

## Adding New Presets

To add a new preset:

1. Add the model option to `openCodeModelOptionMap`
2. Add the preset definition to `openCodePresetMap`
3. Update `models.md`
4. Update `README.md`
5. Update `AGENTS.md`

### Example: Adding a Security Audit Preset

```ts
// In src/lib/opencode/presets.ts

const openCodePresetMap = {
  // ... existing presets
  'security-auditor': {
    id: 'security-auditor',
    label: 'Security Auditor',
    description: 'Security-focused analysis and vulnerability detection.',
    defaultModelOptionId: 'venice-claude-sonnet-4.6',
    provider: 'venice',
    model: 'claude-sonnet-4-6',
    requiredEnv: ['VENICE_API_KEY'],
    agentPrompt: 'Act as a security specialist...',
    instructionsMd: `...`,
    starterPrompt: 'Audit this repository for security vulnerabilities...',
    starterPromptPlaceholder: 'Describe the security concern or code to audit.',
    skills: [
      {
        id: 'buddypie-security-scan',
        name: 'buddypie-security-scan',
        description: 'Scan for common vulnerability patterns.',
        content: `...`,
      },
    ],
    mcp: {},
  },
}
```

## Runtime Behavior

### Model Resolution

When a sandbox launches:

1. Check if `agentProvider` and `agentModel` are specified
2. Validate against supported model options
3. Fall back to preset default if not specified
4. Inject required environment variables
5. Seed the first prompt with an automatic completion sequence so the agent builds, checks types, and pushes the branch without waiting for a follow-up prompt

### Config Generation

The `buildOpenCodeConfig` function generates:

```json
{
  "$schema": "https://opencode.ai/config.json",
  "model": "openrouter/minimax/minimax-m2.7",
  "default_agent": "general-engineer",
  "instructions": [".buddypie/opencode/AGENTS.md"],
  "permission": "allow",
  "agent": {
    "general-engineer": {
      "description": "Balanced repo analysis...",
      "mode": "primary",
      "model": "openrouter/minimax/minimax-m2.7",
      "prompt": "You are running in a Daytona sandbox...",
      "permission": "allow"
    }
  }
}
```

### Permission Model

BuddyPie now starts the OpenCode web agent with blanket `allow` permissions at
both the session and agent levels. Managed skills are still written into the
sandbox, but the web agent is not sandboxed by a BuddyPie-level skill denylist.

## Troubleshooting

<Accordions>
  <Accordion id="model-not-found" title="Model not supported error">
    
    Ensure the model ID matches exactly. OpenRouter uses `{provider}/{model}` format, while Venice uses just the model ID.
    
    Check that the required environment variable is set.
    
  </Accordion>
  
  <Accordion id="preset-load-error" title="Preset fails to load">
    
    Verify the preset ID is valid. Check `isOpenCodeAgentPresetId()` for validation.
    
    Ensure all required environment variables are configured.
    
  </Accordion>
  
  <Accordion id="skill-permission-denied" title="Skill permission denied">
    
    BuddyPie no longer applies a preset-level skill denylist in the generated
    OpenCode config. If a capability is still blocked, inspect the underlying
    OpenCode runtime or sandbox environment rather than the BuddyPie preset.
    
  </Accordion>
</Accordions>
